# Deploy SMS OTP Edge Function

## Overview
The `send-sms-otp` edge function handles SMS OTP verification using BlueDotSMS API for phone number verification during signup.

## Step 1: Deploy the Edge Function

### Via Supabase Dashboard (Recommended)

1. Go to https://supabase.com/dashboard/project/celkdpsuqeqmzevlzkqj/functions
2. Click **"Deploy a new function"** or **"New Edge Function"**
3. Name it: `send-sms-otp`
4. Copy all code from `supabase/functions/send-sms-otp/index.ts`
5. Paste into the editor
6. Click **"Deploy function"**

## Step 2: Set Environment Variables (Secrets)

The edge function requires BlueDotSMS API credentials as environment variables.

### Via Supabase Dashboard:

1. Go to https://supabase.com/dashboard/project/celkdpsuqeqmzevlzkqj/settings/functions
2. Scroll to **"Edge Function Secrets"**
3. Add the following secrets:

| Secret Name | Value | Required |
|-------------|-------|----------|
| `BLUEDOT_API_ID` | `API40426789926` | Yes |
| `BLUEDOT_API_PASSWORD` | `ySxq86umMq` | Yes |
| `BLUEDOT_SENDER_ID` | Leave empty for now | No |

4. Click **"Save"** for each secret

**Note on Sender ID:**
- `BLUEDOT_SENDER_ID` can be left empty initially
- BlueDotSMS will use your account's default sender ID
- Once you register and approve "TodaPay" as a sender ID with BlueDotSMS, you can add it here
- To register a sender ID, contact BlueDotSMS support or use their dashboard

### Via Supabase CLI (Alternative):

```bash
npx supabase secrets set BLUEDOT_API_ID=API40426789926
npx supabase secrets set BLUEDOT_API_PASSWORD=ySxq86umMq
npx supabase secrets set BLUEDOT_SENDER_ID=TodaPay
```

## Step 3: Test the Edge Function

### Test Sending OTP:

```bash
curl -X POST https://celkdpsuqeqmzevlzkqj.supabase.co/functions/v1/send-sms-otp \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_ANON_KEY" \
  -d '{
    "action": "send",
    "phoneNumber": "263712345678",
    "brand": "TodaPay"
  }'
```

```json

  "success": true,
  "verificationId": 12345,
  "message": "Verification Code Sent Successfully"
```

## How It Works

```
User enters details → Click "Send Code"
                 ↓
    Edge Function calls BlueDotSMS Verify API
                 ↓
          SMS sent to user's phone
                 ↓
        User enters OTP code → Click "Verify"
                 ↓
    Edge Function calls BlueDotSMS VerifyStatus API
                 ↓
         OTP verified successfully
                 ↓
    Supabase account created with phone in metadata
```

## Troubleshooting

### Error: "BlueDotSMS credentials not configured"
- **Solution:** Set the environment variables (Step 2)

### Error: "Failed to send OTP"
- **Check:** BlueDotSMS API credentials are correct
- **Check:** Phone number format (should be: 263XXXXXXXXX, no spaces or +)
- **Check:** BlueDotSMS account has sufficient balance

### Error: "Invalid verification code"
- **Check:** User entered correct 6-digit code
- **Check:** Code hasn't expired (typically valid for 10 minutes)
- **Try:** Resend code option
